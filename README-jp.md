## 📖 プロジェクト概要
**Java Security Education Framework (JSEF)** はSpring Boot 3.xをベースに構築されたWebセキュリティ実践プラットフォームで、**開発者、セキュリティリサーチャー、大学生、企業研修**向けに設計されています。**35種類以上の実際のビジネスシナリオに基づくセキュリティ脆弱性事例**（インジェクション攻撃、不正アクセス、機密情報漏洩などのコアタイプを含む）を通じて、「**原理説明→脆弱性再現→コード比較→修復検証**」の完全な学習サイクルを提供し、学習者が「理論」から「実践」へとWebセキュリティのコア能力を迅速に習得するのを助けます。

本プロジェクトは複雑な環境に依存せず、ローカルでのワンクリック起動とDockerデプロイをサポートしています。すべての脆弱性事例は実際のビジネスロジックに基づいて設計されており、「脆弱性のための脆弱性」というデモ用コードを回避し、実際の開発シナリオにより近い形で提供しています。

## 🚀 クイックスタート
### 環境要件
- JDK 17 以上
- Maven 3.6+ または Gradle 8.0+
- Git（オプション、リポジトリのクローン用）
- Docker（オプション、コンテナ化デプロイ用）

### 方法1：ローカルMaven起動（初心者推奨）
```bash
# 1. リポジトリをクローン（または直接ZIPパッケージをダウンロード）
git clone --depth 1 https://github.com/XiaomingX/JSEF.git
cd JSEF

# 2. プロジェクトをビルド（テストをスキップしてビルドを高速化）
mvn clean package -DskipTests

# 3. サービスを起動
java -jar target/springboot-security-sample-0.0.1-SNAPSHOT.jar
```

### 方法2：Dockerワンクリックデプロイ
```bash
# 1. イメージをビルド
docker build -t jsef-security-sample:latest .

# 2. コンテナを起動
docker run -d -p 8080:8080 --name jsef-demo jsef-security-sample:latest
```

### デプロイ成功の検証
起動後、以下のアドレスにアクセスしてください：
- プロジェクトホームページ：`http://localhost:8080`（プロジェクトナビゲーションと脆弱性リストを確認）
- APIドキュメント（Swagger）：`http://localhost:8080/swagger-ui.html`（すべての脆弱性インターフェースの詳細を確認）
- 脆弱性マニュアル：`http://localhost:8080/docs`（オンラインの脆弱性再現ガイドを確認）


## 📋 脆弱性事例分類（35種類以上の完全リスト）
<details>
<summary>クリックして完全な脆弱性分類を展開（OWASP Top 10を全てカバー）</summary>

### 1. インジェクション系脆弱性
- SQLインジェクション：基本的な文字列結合インジェクション、エラーベースインジェクション、ブラインドインジェクション、プリペアドステートメントの比較事例
- コマンドインジェクション：Runtime.exec()の誤用、ProcessBuilderインジェクションシナリオ
- テンプレートインジェクション：FreeMarker/Thymeleaf/Velocityインジェクション事例
- SPELインジェクション：Spring Expression Languageインジェクション脆弱性と防御策
- XSS（クロスサイトスクリプティング）：リフレクテッドXSS、ストアドXSS、DOMベースXSS（CSP防御デモを含む）
- LDAPインジェクション：ディレクトリサービスクエリのインジェクションシナリオと防御策
- XML外部エンティティ（XXE）：XMLパーサーの不適切な設定による情報漏洩

### 2. 認証・認可系脆弱性
- 認証バイパス：Cookie偽造、セッション固定攻撃
- 不正アクセス（特権昇格）：水平方向の不正アクセス（ユーザー間のデータアクセス）、垂直方向の不正アクセス（低権限ユーザーによる管理者インターフェースへのアクセス）
- 弱いパスワードのリスク：平文パスワード検証、パスワード複雑性のバイパス
- JWT脆弱性：署名バイパス、有効期限の改ざん、シークレットキーの漏洩
- セッション管理の不備：不適切なセッションタイムアウト設定、セッションIDの暴露

### 3. 機密情報漏洩
- 平文伝送：暗号化されていないHTTPによるCookie/Tokenの漏洩
- エラーページによる情報漏洩：スタックトレースの暴露、設定情報の漏洩
- ログによる情報漏洩：機密データ（電話番号、身分証明書番号）の平文ロギング
- サードパーティ依存関係の漏洩：依存コンポーネントのバージョン暴露（CVE-2023-20860などの事例を含む）
- パスワードの不適切な保存：平文保存、弱いハッシュアルゴリズム（MD5/SHA1）の使用

### 4. 不適切な設定
- デフォルトパスワードのリスク：変更されていない管理者デフォルトパスワード
- 不安全なHTTPメソッド：認可なしでPUT/DELETEメソッドへのアクセスを許可
- CORSの不適切な設定：過度に緩いクロスオリジンリソース共有（CORS）設定
- キャッシュメカニズムの脆弱性：機密ページのキャッシュによる情報漏洩
- セキュリティレスポンスヘッダーの欠落：CSP、X-Frame-Optionsなどの保護ヘッダーの不足

### 5. その他の高リスク脆弱性
- ファイルアップロード脆弱性：ファイル拡張子のバイパス、MIMEタイプの偽装、ファイルコンテンツ解析の脆弱性
- パストラバーサル：ディレクトリトラバーサルによるシステムファイル（/etc/passwdなど）の読み取り
- 逆シリアライズ脆弱性：Jackson/Gsonの逆シリアライズによるリモートコード実行
- 依存関係混同攻撃：サプライチェーン攻撃のデモ（依存関係ハイジャック事例を含む）
- サーバーサイドリクエストフォージェリ（SSRF）：内部サービスへのアクセスとデータ窃取
- 逆シリアライズ脆弱性：Javaシリアライズ/逆シリアライズメカニズムの誤用
</details>

1. **Issueの投稿**：脆弱性のフィードバック、機能提案、バグ報告を行う（事前に類似のIssueがないか検索することを推奨）
2. **PR（プルリクエスト）の投稿**：
   - コードの問題修正（スペルミス、ロジック最適化など）
   - 新しい脆弱性事例の追加（[新しい脆弱性事例の追加ガイド](docs/contribute-vulnerability.md)に従う必要があります）
   - ドキュメントの充実（再現手順の追加、英文ドキュメントの翻訳など）
3. **シェアと普及**：本プロジェクトにStarをつけ、技術コミュニティで使用体験を共有して、更多の人にJSEFを知ってもらう
